FAQ

常见问题

如何用 npm 安装 CLI `os`？

全局安装：npm install -g opensunstar-os（需 Node.js 18+）。包名 opensunstar-os 为 Node 薄包装，安装时从 GitHub Release 下载对应平台的 Rust 原生二进制。验证：os --version。也可从 Releases 手动下载 os-* 附件。

CLI `os` 需要先启动桌面 GUI 吗？

不需要。 os 可单独下载运行，首次使用会自动创建 ~/.OpenSunstar/OpenSunstar.db（或执行 os config bootstrap --yes）。治理、供应商切换、TUI 仪表盘均不依赖 GUI。若使用代理接管（本地 :15721），请保持桌面应用运行。

切换供应商后需要重启终端吗？

大多数 CLI 需要重启终端才能生效。Claude Code 支持供应商热切换，无需重启。切换前 OpenSunstar 会自动备份当前配置。

我的数据存储在哪里？

供应商、MCP、Prompts、Skills 存储在 ~/.OpenSunstar/OpenSunstar.db（SQLite）。UI 偏好设置在 settings.json，自动备份保留最近 10 份。

如何切回官方登录？

在预设供应商中添加「官方（Official）」条目并切换过去，然后在对应 CLI 中执行 Log out / Log in 流程即可。

项目组合和看板有什么区别？

OpenSunstar 的「项目组合」是多 Git 仓库研发仪表盘，展示代码量、提交活跃度与 AI 洞察，不是拖拽式任务看板。

OpenSunstar 安全吗？会上传我的数据吗？

OpenSunstar 全本地运行，零遥测，不上传任何数据。API Key 加密存入操作系统密钥链（Windows Credential Manager / macOS Keychain / Linux Secret Service），使用 AES-256-GCM 加密。Skills 安装前经过 55+ 规则安全审计，确保来源可信。

如何在不同设备间同步配置？

支持 WebDAV、S3、GitHub Gist 三种云同步通道。也可以将配置目录指向 Dropbox、OneDrive 或 iCloud 实现跨设备同步。所有配置自动备份，保留最近 10 份快照，随时可恢复。也可导出 SQLite 文件手动迁移。
